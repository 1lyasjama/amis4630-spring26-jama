## 5. Code Quality Coaching (Non-Scoring)

- **Missing `async` in LINQ query**: [JwtTokenService.cs](api/Services/JwtTokenService.cs#L82) — `RotateRefreshTokenAsync` uses synchronous `FirstOrDefault` instead of `FirstOrDefaultAsync` for the database lookup. This blocks the thread; switching to `await _db.RefreshTokens.FirstOrDefaultAsync(...)` would be more consistent with the async pattern used everywhere else.

- **Confirm dialog for delete**: [AdminProductsPage.tsx](frontend/src/pages/AdminProductsPage.tsx#L73) — uses `confirm()` (browser built-in) for delete confirmation. A custom modal component would be more polished and testable, though the current approach is functional.

- **Token expiry check on page load only**: [AuthContext.tsx](frontend/src/context/AuthContext.tsx#L17-L21) — the stored user expiry is checked only when the context initializes. A periodic check or an Axios/fetch interceptor that proactively refreshes before expiry would prevent edge cases where the token expires mid-session without a 401 trigger.

- **`GetOrderById` authorization gap**: [OrdersController.cs](api/Controllers/OrdersController.cs#L76-L86) — when a non-admin user requests an order that belongs to another user, the endpoint returns `NotFound` instead of `Forbidden`. Returning 404 is actually a reasonable security practice (avoids leaking order ID existence), but be aware this is a deliberate design choice worth documenting.
