# Playwright MCP — E2E Run Notes

This document records how the Milestone 5 happy-path E2E spec was generated by
the agent, what it failed on the first attempt, and what was corrected.

## Spec under test

`frontend/e2e/checkout.spec.ts` — single test that walks the full shopper
journey: register a fresh user, log out, log back in, browse the catalog, add
the first product to the cart, proceed to checkout, submit a shipping address,
and verify the order shows up in `/orders`.

## How to run

From the `frontend/` directory:

```bash
npx playwright install chromium   # one-time, downloads browser binaries
CI=1 npx playwright test          # runs the spec; webServer config auto-starts
                                  # the .NET API on :5023 and Vite on :5173
```

`playwright.config.ts` sets `ASPNETCORE_ENVIRONMENT=Development` for the API
process so `dotnet user-secrets` (Jwt:Key, etc.) is loaded.

## Prompts given to the agent

1. **Initial generation.** "Write a single Playwright spec at
   `frontend/e2e/checkout.spec.ts` that covers the Milestone 5 happy path:
   register a brand-new unique user, log out, log back in, add the first
   product to the cart, proceed to checkout, fill in a shipping address, place
   the order, and assert that the order appears in `/orders`. Use accessible
   role/label selectors so the test is resilient to CSS changes."

2. **Webserver setup.** "Configure `playwright.config.ts` so `npx playwright
   test` boots both the .NET API (`dotnet run --project ../api --urls
   http://localhost:5023`) and the Vite dev server (`npm run dev -- --port
   5173 --strictPort`), with `ASPNETCORE_ENVIRONMENT=Development` so that
   user-secrets are loaded."

3. **Selector fix-ups.** "Update the spec to match the actual DOM: the
   checkout link reads 'Proceed to Checkout' and the logout control is a
   `<button>`, not a link."

## What failed on the first run, and the corrections

| # | Failure | Root cause | Fix |
|---|---------|------------|-----|
| 1 | API startup crashed with `Jwt:Key is not configured` when launched from the Playwright `webServer` block. | `--no-launch-profile` skips `launchSettings.json`, so `ASPNETCORE_ENVIRONMENT` defaulted to `Production` and user-secrets were never loaded. | Added `env: { ASPNETCORE_ENVIRONMENT: 'Development' }` to the API `webServer` entry in `playwright.config.ts`. |
| 2 | `getByRole('link', { name: /logout/i })` timed out after registration. | The header renders Logout as `<button>` styled like a link, not as `<a>`. | Switched the selector to `getByRole('button', { name: /logout/i })`. |
| 3 | `getByRole('link', { name: /checkout/i })` ambiguous / mis-targeted. | The actual cart-summary link text is "Proceed to Checkout". | Tightened the regex to `/proceed to checkout/i`. |
